# 追剧日历管家

管理所有在追剧集，自动追踪更新进度、预测完结时间、生成日程表，并在剧荒时智能推荐。

## 默认模式：追剧管理

当用户提到与追剧相关的话题时，自动激活此技能。根据用户输入内容，自动判断意图并调用相应的脚本。

**触发示例：**
- "我在看《黑暗荣耀》，更新到第8集" → 录入进度
- "本周追剧日历" → 生成日程
- "剧荒了，推荐几部" → 智能推荐
- "《繁花》什么时候完结" → 完结预测
- "《黑暗荣耀》第7集讲了什么" → 前情提要

| 用户意图 | 用户说一句话 | 调用的脚本 |
|----------|--------------|------------|
| 录入进度 | "我在看《黑暗荣耀》，更新到第8集" | parser.py |
| 查看日程 | "本周追剧日历" | scheduler.py |
| 完结预测 | "《繁花》什么时候完结" | predictor.py |
| 智能推荐 | "剧荒了，推荐几部" | recommender.py |
| 前情提要 | "《庆余年2》第20集讲了什么" | recap.py |

## 执行逻辑说明

本技能采用「意图识别 → 脚本调度 → 数据操作 → 格式化输出」四步执行架构。

```
用户自然语言输入
    ↓
步骤1：意图识别（判断用户想做什么）
    ↓
步骤2：脚本调度（调用对应的 scripts/ 脚本）
    ↓
步骤3：数据操作（通过 data_manager.py 读写 data/drama_data.json）
    ↓
步骤4：格式化输出（返回结构化的中文结果）
```

### 步骤1：意图识别

**执行文件：** 本步骤在 SKILL.md 中由 Claude 直接执行，不调用外部脚本。

**执行内容：**
获取用户输入的自然语言文本，判断用户意图属于以下五类之一：

| 意图类型 | 触发关键词示例 | 对应脚本 |
|----------|---------------|----------|
| 追剧录入 | "我在看"、"更新到"、"看到第X集"、"开始看" | parser.py |
| 日程生成 | "本周追剧日历"、"这周看什么"、"追剧日程" | scheduler.py |
| 完结预测 | "什么时候完结"、"还有多久"、"预测" | predictor.py |
| 智能推荐 | "剧荒"、"推荐"、"有什么好看的" | recommender.py |
| 前情提要 | "第X集讲了什么"、"前情提要"、"上一集" | recap.py |

**处理逻辑：**
1. 扫描用户输入中是否包含上述触发关键词。
2. 若包含单一意图关键词，直接进入对应脚本调度。
3. 若包含多个意图关键词，按优先级排序（录入 > 回顾 > 预测 > 日程 > 推荐）。
4. 若无法识别意图，返回引导提示："请问您是想录入追剧进度、查看日程、预测完结、获取推荐还是回顾剧情？请告诉我具体需求。"
5. 若识别为不适用场景（如影评撰写、剧本创作），返回提示："本技能仅支持追剧管理相关操作，不支持影视剧创作、剧本分析或影评撰写。"

### 步骤2：脚本调度

根据步骤1识别的意图，调用对应的脚本文件。

#### 场景A：追剧录入（意图识别结果 → parser.py）

**执行文件：** `scripts/parser.py`

**调用方式：**
```bash
python scripts/parser.py "《黑暗荣耀》我看到第8集，最新16集"
```

**执行内容：**
- 接收用户输入的自然语言文本。
- 使用正则表达式匹配以下模式：
  - 剧名提取：匹配《》或「」内的内容
  - 我的集数提取：匹配"我看到第X集"、"我X集"、"X/Y格式"等
  - 最新集数提取：匹配"最新X集"、"更新到X集"、"X/Y格式"等
- 若用户输入为"开始看《剧名》"且未提供集数，默认我的集数=1，最新集数=1。
- 若解析失败，返回格式提示："请按格式输入，示例：《黑暗荣耀》我看到第8集，最新16集"
- 解析成功后，调用 data_manager.py 的 `add_or_update_drama()` 函数保存数据。

#### 场景B：日程生成（意图识别结果 → scheduler.py）

**执行文件：** `scripts/scheduler.py`

**调用方式：**
```bash
python scripts/scheduler.py
```

**执行内容：**
- 调用 data_manager.py 的 `get_all_dramas()` 函数读取 data/drama_data.json 中所有在追剧集。
- 若数据文件为空，返回提示："暂无追剧记录，请先录入追剧进度。"
- 从 references/schedule_rules.yaml 读取各剧的播出规律配置（更新日、每周更新集数）。
- 按周一至周日分配每部剧的下一集更新日期。
- 统计本周更新总集数。
- 格式化为结构化日程表输出。

#### 场景C：完结预测（意图识别结果 → predictor.py）

**执行文件：** `scripts/predictor.py`

**调用方式：**
```bash
python scripts/predictor.py "黑暗荣耀"
```

**执行内容：**
- 从用户输入中提取剧名（支持《》包裹或无书名号）。
- 若未提供剧名，返回提示："请告诉我剧名，例如：'《黑暗荣耀》什么时候完结'"
- 调用 data_manager.py 的 `get_drama_info()` 函数读取该剧的当前进度（my_episode 和 latest_episode）。
- 若该剧不在数据中，返回提示："您还没有录入《剧名》的追剧进度，请先录入。"
- 计算剩余集数 = latest_episode - my_episode。
- 按每天追1集的速度推算完结日期：预计完结日期 = 当前日期 + 剩余集数天。
- 根据剩余集数生成个性化建议：
  - 剩余 ≤ 3 集 → "快追完了，加油！"
  - 剩余 ≤ 7 集 → "一周内能追完，坚持！"
  - 剩余 > 7 集 → "还有 X 集，慢慢享受"
- 格式化为预测报告输出。

#### 场景D：智能推荐（意图识别结果 → recommender.py）

**执行文件：** `scripts/recommender.py`

**调用方式：**
```bash
python scripts/recommender.py
```

**执行内容：**
- 调用 data_manager.py 的 `get_all_dramas()` 函数读取所有在追剧集。
- 若数据文件为空，返回提示："暂无追剧记录，请先录入追剧进度，我才能为您推荐。"
- 分析用户偏好：
  - 从追剧历史中提取高频类型标签（悬疑、古装、年代等）
  - 计算追剧速度（待更新集数 ÷ 在追部数）
- 从内置推荐池（20部剧）中筛选：
  - 过滤用户已追过的剧
  - 按类型匹配度、评分、年份、集数多维度评分
- 排序取前5名。
- 为每部推荐剧生成个性化推荐理由。
- 格式化为推荐列表输出。

#### 场景E：前情提要（意图识别结果 → recap.py）

**执行文件：** `scripts/recap.py`

**调用方式：**
```bash
python scripts/recap.py "黑暗荣耀" 7
```

**执行内容：**
- 从用户输入中提取剧名和集数。
- 若未提供剧名或集数，返回提示："请提供剧名和集数，例如：'《黑暗荣耀》第7集讲了什么'"
- 调用 data_manager.py 的 `get_drama_info()` 函数读取该剧当前进度。
- 查询预设摘要数据（MOCK_RECAPS 字典）：
  - 若存在该剧该集的预设摘要，直接使用
  - 若不存在，AI 动态生成摘要
- 格式化输出：追剧进度 + 主要事件 + 关键台词 + 本集伏笔 + 继续观看提示。

### 步骤3：数据操作

**执行文件：** `scripts/data_manager.py`

**核心作用：** 所有脚本的数据读写统一接口，确保数据一致性。

**数据文件：** `data/drama_data.json`

**提供的数据接口：**

| 函数名 | 功能说明 | 被谁调用 | 执行操作 |
|--------|----------|----------|----------|
| load_data() | 从 data/drama_data.json 加载数据 | 内部函数 | 读取 JSON 文件，返回字典 |
| save_data() | 保存数据到 data/drama_data.json | 内部函数 | 将字典写入 JSON 文件 |
| add_or_update_drama() | 添加或更新追剧记录 | parser.py | 检查剧名是否存在，新增或更新集数 |
| get_all_dramas() | 获取所有在追剧集 | scheduler.py, predictor.py, recommender.py | 返回 watch_list 字典 |
| get_drama_info() | 获取单部剧信息 | predictor.py, recap.py | 返回指定剧名的进度信息 |
| get_summary() | 获取追剧汇总 | parser.py | 返回格式化的追剧概览 |

**数据格式：**

```json
{
  "watch_list": {
    "黑暗荣耀": {
      "my_episode": 8,
      "latest_episode": 16,
      "last_update": "2026-07-12 14:30"
    }
  },
  "history": [
    {
      "drama_name": "黑暗荣耀",
      "action": "新增",
      "my_episode": 8,
      "latest_episode": 16,
      "date": "2026-07-12 14:30"
    }
  ],
  "total_count": 1
}
```

### 步骤4：格式化输出

**执行文件：** 各脚本内置输出格式化逻辑。

**执行内容：**
根据不同的操作类型，返回不同格式的结构化输出。

| 操作类型 | 输出格式要求 | 示例 |
|----------|-------------|------|
| 追剧录入 | 录入确认 + 追剧概览 | "✅ 已添加：《黑暗荣耀》看到第8集，最新第16集。共 2 部在追，累计落后 26 集。" |
| 日程生成 | 日期 + 剧名 + 集数 + 统计 | "📅 本周追剧日历\n【周一】《黑暗荣耀》第 9 集\n共 3 部在追" |
| 完结预测 | 当前进度 + 剩余集数 + 预计日期 + 建议 | "剩余 8 集，预计 2026-07-20 追完。还有 8 集，慢慢享受。" |
| 智能推荐 | 剧名 + 类型 + 评分 + 理由 | "🎬 《狂飙》犯罪悬疑 8.5分。理由：你喜欢悬疑类型。" |
| 前情提要 | 进度 + 事件 + 台词 + 伏笔 | "📝 主要事件：文东恩发现河道英的真实身份..." |

## 脚本说明

### 脚本总览表

| 脚本 | 功能 | 调用方式 | 是否独立运行 |
|------|------|----------|--------------|
| data_manager.py | 数据读写核心 | 被其他5个脚本引用 | 是 |
| parser.py | 自然语言解析与录入 | `python parser.py "用户输入"` | 是 |
| scheduler.py | 生成周日程表 | `python scheduler.py` | 是 |
| predictor.py | 预测完结日期 | `python predictor.py "剧名"` | 是 |
| recommender.py | 智能推荐5部剧 | `python recommender.py` | 是 |
| recap.py | 生成前情提要 | `python recap.py "剧名" 集数` | 是 |

### 脚本1：data_manager.py（数据管理核心）

**代码作用：** 所有脚本的数据读写核心，提供统一的数据接口。负责创建、读取、更新、删除追剧数据。

**怎么用：** 被其他5个脚本引用，不直接由用户运行。

**使用示例：**

```python
from data_manager import add_or_update_drama, get_all_dramas, get_drama_info

# 添加或更新追剧记录
add_or_update_drama("黑暗荣耀", 8, 16)

# 获取所有在追剧集
all_dramas = get_all_dramas()

# 获取单部剧信息
drama_info = get_drama_info("黑暗荣耀")
print(drama_info)  # {'my_episode': 8, 'latest_episode': 16}
```

**提供的数据接口：**

| 函数名 | 功能 | 被谁调用 |
|--------|------|----------|
| load_data() | 从 data/drama_data.json 加载数据 | 内部函数 |
| save_data() | 保存数据到 data/drama_data.json | 内部函数 |
| add_or_update_drama() | 添加或更新追剧记录 | parser.py |
| get_all_dramas() | 获取所有在追剧集 | scheduler.py, predictor.py, recommender.py |
| get_drama_info() | 获取单部剧信息 | predictor.py, recap.py |
| get_summary() | 获取追剧汇总 | parser.py |

### 脚本2：parser.py（追剧录入）

**代码作用：** 解析用户的自然语言输入，提取剧名、我的集数、最新集数，并自动保存到数据文件。

**怎么用：** 用户通过命令行直接运行。

**支持的自然语言格式：**
- 《剧名》我看到第X集，最新X集
- 《剧名》我X集，更新X集
- 剧名 我X 最新X
- 剧名 X/Y（X是我的集数，Y是最新集数）
- 开始看《剧名》（默认从第1集开始）

### 脚本3：scheduler.py（日程生成）

**代码作用：** 读取所有追剧数据，按周分配更新日，生成结构化的本周追剧日程表。

### 脚本4：predictor.py（完结预测）

**代码作用：** 读取指定剧的当前进度，计算剩余集数，按每天1集的速度推算完结日期，并给出个性化建议。

### 脚本5：recommender.py（智能推荐）

**代码作用：** 基于用户追剧数据，分析偏好类型、追剧速度，从推荐池中多维度评分，推荐5部可能喜欢的剧。

### 脚本6：recap.py（前情提要）

**代码作用：** 读取追剧进度，查询或动态生成指定剧集的内容摘要，结构化输出。

## 输出风格与沟通原则

- 面向追剧用户，使用清晰、友好、不啰嗦的中文描述，避免过度修饰。
- 对于不确定的推断（如预测完结时间），使用「预计」「可能」等措辞，并说明依据。
- 把「数据」与「建议」区分开：进度数据在前，追剧建议在后。
- 如发现用户输入不明确或数据不足，应该坦诚指出，并建议下一步操作。

## 常见错误与纠正策略

**错误：用户输入模糊，无法识别意图。**

纠正：向用户提出具体问题，引导提供剧名。

回复示例："请问您说的是哪部剧？请告诉我剧名，我来帮您查询进度。"

---

**错误：数据文件不存在，但操作需要读取数据。**

纠正：提示用户先录入追剧数据，再查看日程。

回复示例："您还没有追剧记录，请先录入再查看日程。"

---

**错误：解析自然语言失败，无法提取剧名或集数。**

纠正：提示用户使用支持的格式，并提供示例。

回复示例："我没能理解您的意思。请这样告诉我：'我在看《黑暗荣耀》，更新到第8集'"

---

**错误：用户要查的剧不在数据中。**

纠正：提示该剧不在追剧清单中，引导用户先录入。

---

**错误：用户要看的集数超出当前进度。**

纠正：提示用户该剧已完结或集数不存在，给出正确的集数范围。

## 故障排除

**错误："找不到 SKILL.md 文件"**
- 原因：文件缺失或命名错误（如 SKILL.MD、skill.md 等）
- 解决：在 skill 文件夹根目录创建 SKILL.md（注意大小写）

**错误："YAML 解析失败"**
- 原因：frontmatter 中 YAML 语法无效
- 解决：检查缺少 --- 分隔符、缩进不正确、描述中包含未转义的特殊字符

**错误："名称字段无效"**
- 原因：名称包含空格、大写字母或特殊字符
- 解决：使用短横线命名（例如 my-skill-name）

**警告："SKILL.md 文件太长"**
- 原因：文件超过 300 行
- 解决：将详细内容移至 references/ 文件夹
