# 追剧日历管家 - Skill 定义检查清单

本文档用于检查 Skill 定义是否符合最佳实践。基于 Skill Reviewer 的 20 项检查清单。

---

## 类别一：结构（Structure）

| 编号 | 检查项 | 说明 | 状态 |
|------|--------|------|------|
| S1 | SKILL.md 存在 | 技能根目录下必须有 SKILL.md 文件 | ✅ |
| S2 | 目录结构规范 | 包含 scripts/ 和 references/ 目录 | ✅ |
| S3 | 脚本文件存在 | scripts/ 下所有脚本文件存在且可执行 | ✅ |
| S4 | 参考文件存在 | references/ 下配置文件完整 | ✅ |

**结构检查说明：**

- **S1**：SKILL.md 是技能的入口文件，Claude 首先读取它来理解技能功能。文件必须放在技能根目录下，命名必须完全匹配（区分大小写）。
- **S2**：scripts/ 目录存放可执行脚本，references/ 目录存放配置和参考文档。这是 Anthropic 官方推荐的标准结构。
- **S3**：所有在 SKILL.md 中引用的脚本文件必须真实存在，且包含可执行的入口点（如 `if __name__ == "__main__"`）。
- **S4**：references/ 中的配置文件（如 schedule_rules.yaml、sample_history.json）必须存在且格式正确。


## 类别二：格式（Format）

| 编号 | 检查项 | 说明 | 状态 |
|------|--------|------|------|
| F1 | YAML frontmatter | 文件以 --- 开头，包含 YAML 元数据 | ✅ |
| F2 | name 字段 | 使用短横线命名（小写+连字符） | ✅ |
| F3 | description 字段 | 包含触发条件和使用场景 | ✅ |
| F4 | version 字段 | 语义化版本号（如 2.1.0） | ✅ |
| F5 | author 字段 | 标明作者 | ✅ |

**格式检查说明：**

- **F1**：YAML frontmatter 必须位于文件最顶部，以 `---` 开头和结尾。Claude 使用这些元数据来识别技能的属性和触发条件。
- **F2**：name 字段应使用短横线命名（如 `my-skill-name`），避免使用空格、大写字母或特殊字符。
- **F3**：description 字段应简洁描述技能功能，并包含触发关键词，帮助 Claude 判断何时调用此技能。
- **F4**：version 字段使用语义化版本号（主版本.次版本.补丁版本），便于追踪技能的迭代历史。
- **F5**：author 字段标明技能作者，便于责任追溯和沟通。


## 类别三：内容（Content）

| 编号 | 检查项 | 说明 | 状态 |
|------|--------|------|------|
| C1 | 指令清晰可执行 | Claude 能理解并执行 | ✅ |
| C2 | 有具体示例 | 包含输入输出示例 | ✅ |
| C3 | 输出格式明确 | 每种操作的输出格式有示例 | ✅ |
| C4 | 错误处理说明 | 包含常见错误及处理方式 | ✅ |
| C5 | 脚本引用关系清晰 | 说明脚本间调用关系 | ✅ |
| C6 | 不重复已知信息 | 只写 Claude 不知道的 | ✅ |
| C7 | 简洁精炼 | SKILL.md 不超过 300 行 | ✅ |
| C8 | 渐进式披露 | 详细内容放在 references/ | ✅ |

**内容检查说明：**

- **C1**：每条指令都应具体明确，Claude 能够直接理解并执行，无需额外解释。
- **C2**：每个功能都应提供具体的输入输出示例，帮助 Claude 理解使用场景。
- **C3**：不同操作类型的输出格式应有明确示例（如日程表、预测报告、推荐列表等）。
- **C4**：应说明常见错误场景及纠正策略，帮助 Claude 在遇到问题时正确引导用户。
- **C5**：应清晰说明各脚本之间的引用关系（如 data_manager.py 被其他脚本引用）。
- **C6**：只写 Claude 不知道的信息（如脚本路径、调用方式），不解释基础概念。
- **C7**：SKILL.md 应保持简洁，控制在 300 行以内，便于 Claude 快速读取。
- **C8**：详细内容（如完整检查清单、报告模板、分析维度）应放在 references/ 目录。


## 类别四：触发（Trigger）

| 编号 | 检查项 | 说明 | 状态 |
|------|--------|------|------|
| T1 | 触发词有效 | description 中的触发词能正确匹配 | ✅ |
| T2 | 触发词数量合理 | 不超过 5 个核心触发词 | ✅ |
| T3 | 触发条件明确 | 什么情况下激活 Skill 描述清晰 | ✅ |

**触发检查说明：**

- **T1**：description 中的触发词应与用户可能的输入相匹配，确保 Claude 能正确识别并调用技能。
- **T2**：核心触发词应控制在 5 个以内，避免过度触发或误触发。
- **T3**：应明确说明技能的使用边界（什么场景用、什么场景不用），避免 Claude 在不合适的场景下调用。
